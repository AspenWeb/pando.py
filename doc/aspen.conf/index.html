Aspen can be configured via an aspen.conf file. This is an INI-style file per the standard library's RawConfigParser object. It is read from the following four locations. Later files override earlier.

  • /etc/aspen/aspen.conf
  • /usr/local/etc/aspen/aspen.conf
  • ~/aspen/aspen.conf
  • ./.aspen/aspen.conf

[aspen]

Aspen uses the following keys from the [aspen] section of aspen.conf.

default_filenames

When given a request for a directory, Aspen will look for the files named here to serve the request. The first one found will be used.

Valid values: comma- and/or space-separated list of strings
Default: index.html
Example: 
[aspen]
default_filenames: index.html, default.html, README
default_mimetype

If Aspen can’t guess a mimetype based on the filename extension, it will use this value. You might set this to application/octet-stream, for example, to conform better with the HTTP spec.

Valid values: any string
Default: text/plain
json_content_type

Aspen has special support for JSON simplates. Use this key to preserve special JSON simplate handling but set a different Content-Type header.

Valid values: any string
Default: application/json
Example: 
[aspen]
json_content_type: text/plain
list_directories

If set to yes, Aspen will show a directory listing when none of the files named by default_filenames exist in a given directory. If set to no, requests for such directories will result in a 404.

Valid values: yes/no/true/false/0/1
Default: no
show_tracebacks

If set to yes, Aspen will display debugging information for error responses. If set to no, a simple HTTP message will be shown without additional detail.

Valid values: yes/no/true/false/0/1
Default: no

[aspen.cli]

The aspen command line program uses the following keys from the [aspen.cli] section of aspen.conf.

address

This is the IPv4 or Unix socket to listen on.

Valid values: AF_INET (host:port) or AF_UNIX (/path/to/my.sock) socket specifier
Default: 0.0.0.0:8080
changes_kill

If set to yes, Aspen will exit when Python module files or any files under a .aspen hidden directory in the website root change. Use this option with thrash to bounce aspen in development.

Valid values: yes/no/true/false/0/1
Default: no
